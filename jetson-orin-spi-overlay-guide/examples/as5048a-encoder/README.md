# AS5048A Magnetic Encoder Overlay for Jetson Orin

Enables SPI on the 40-pin header so the AS5048A 14-bit magnetic rotary
encoder can be driven from Python.

The overlay configures the Tegra pads `spi1_mosi_pz5` / `spi1_miso_pz4`
/ `spi1_sck_pz3` / `spi1_cs0_pz6` (header pins 19/21/23/24) to their
SPI function. Note the Linux device node depends on the carrier:

* **Jetson Orin Nano Devkit (P3768)** — these pads route to Tegra
  `spi@3210000` and appear as **`/dev/spidev0.0`** (verified by the
  loopback test in `tests/encoder/00_spi_connectivity.py`).
* On some other carriers the same pads route to `spi@3230000` =
  `/dev/spidev1.0`. Run the loopback test to confirm which bus your
  board uses, and set `AS5048AConfig(bus=...)` accordingly.

This overlay only configures the four SPI pins; the encoder's CSN is
driven by hardware CS0, so no extra GPIOs are needed.

## Quick start

```bash
chmod +x install.sh
sudo ./install.sh
# reboot when prompted
```

After reboot:

```bash
ls -l /dev/spidev0.0                                  # node should exist (Orin Nano Devkit)
sudo python3 ../../../tests/encoder/00_spi_connectivity.py
sudo python3 ../../../tests/encoder/01_read_angle.py
```

The connectivity test runs a SPI loopback on both `/dev/spidev0.0`
and `/dev/spidev1.0` to tell you which one the 40-pin header maps to
on your board.

## Wiring

| AS5048A | Jetson pin | Signal              | Tegra pad      |
|---------|------------|---------------------|----------------|
| 3V3     | 17         | 3.3 V               | —              |
| GND     | 25         | GND                 | —              |
| MOSI    | 19         | SPI1 MOSI           | spi1_mosi_pz5  |
| MISO    | 21         | SPI1 MISO           | spi1_miso_pz4  |
| SCK     | 23         | SPI1 SCK            | spi1_sck_pz3   |
| CSN     | 24         | SPI1 CS0            | spi1_cs0_pz6   |

## Files

- `jetson-orin-as5048a.dts` — overlay source (SPI1 pinmux only)
- `install.sh` — compile + install + add `AS5048A` boot entry to
  `/boot/extlinux/extlinux.conf` and set it as default
- `jetson-orin-as5048a.dtbo` — generated by `install.sh`

## Matching Python config

`src/encoder_config.py` ships with the default:

```python
AS5048AConfig(bus=0, device=0, max_hz=1_000_000, mode=1)
```

which targets `/dev/spidev0.0`, SPI mode 1, 1 MHz — compatible with the
AS5048A datasheet (MOSI sampled on falling edge, MISO updated on rising
edge; up to 10 MHz). If the loopback test shows the encoder responds on
`/dev/spidev1.0` instead, change `bus=1`.

## Reverting

Edit `/boot/extlinux/extlinux.conf` and change:

```
DEFAULT AS5048A   ->   DEFAULT primary
```

then reboot. The backup file `extlinux.conf.backup.<timestamp>` created
by the installer can also be restored verbatim.
